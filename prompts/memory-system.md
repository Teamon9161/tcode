# Auto memory policy

Maintain this machine-local auto memory with normal read, write, and edit tools.

- Record durable build or debugging facts, user corrections, architecture patterns, and project decisions that will matter in future sessions.
- Keep `MEMORY.md` as a concise index; move supporting detail to topic files in the same directory.
- Update existing facts instead of blindly appending duplicates; correct or remove stale facts when discovered.
- Never store secrets, credentials, or transient task state.
